API

The public interface of FastBroadcast.jl consists of the @.. macro and the Serial and Threaded threading-dispatch types.

FastBroadcast.Serial — Type
Serial()

Threading dispatch type for @... When passed as thread = Serial(), uses serial broadcasting (the default). Using a type instead of false allows compile-time dispatch.

Example

using FastBroadcast

destination = zeros(3)
source = [1, 2, 3]
@.. thread = Serial() destination = source + 1
source
FastBroadcast.Threaded — Type
Threaded()

Threading dispatch type for @... When passed as thread = Threaded(), enables Polyester-based multithreaded broadcasting. Using a type instead of true allows compile-time dispatch and prevents invalidations when Polyester is loaded (the Serial code path never references threaded methods).

Example

using FastBroadcast, Polyester

destination = zeros(3)
source = [1, 2, 3]
@.. thread = Threaded() destination = source + 1
source
FastBroadcast.@.. — Macro
@.. [thread = false] [broadcast = false] expr

@.. turns expr into a broadcast-like expression, similar to @.. It additionally provides two optional keyword arguments:

  • thread: Defaults to false. Controls multithreading via Polyester.jl. Accepts Serial(), Threaded(), false, or true. Using the type-based Serial()/Threaded() is preferred as it enables compile-time dispatch and avoids invalidations when Polyester is loaded. Requires using Polyester for threaded execution. This only works if broadcast=false.
  • broadcast: Defaults to false. If true, it will broadcast axes with dynamic runtime sizes of 1 to larger sizes, if false only sizes known to be 1 at compile time will be supported, i.e. axes such that ArrayInterface.known_length(typeof(semi_static_axes(x,i))) == 1 will be broadcast. Note that this differs from base broadcasting, in that base broadcasting only supports broadcast=true.

Example

using FastBroadcast

destination = zeros(3)
source = [1, 2, 3]
@.. destination = 2source + 1
source
